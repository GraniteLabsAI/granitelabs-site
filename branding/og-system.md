# 🧱 GraniteLabs OG Cube System

## 🎯 Purpose
This document defines the official Open Graph (OG) image design system for GraniteLabs.

All OG images must follow this standard to ensure visual consistency across the platform.

---

## 🎨 Core Visual Identity

### Background
- Dark granite / stone texture
- Deep blue overlay tone
- Subtle vignette (edges darker)
- Faint circuit / tech lines (low opacity)
- Cyan/blue glow behind centerpiece

---

## 🧱 Centerpiece (Required)

### Granite Cube
- 3D cube with stone texture
- Matches background material family
- Cyan edge lighting (subtle, sharp)
- Soft ground glow beneath cube
- Slight perspective (not flat)

---

## 🔷 Cube Face Content

Each OG image uses ONE symbol on the front face:

### Rules
- Cyan glow style
- Clean, minimal design
- No text inside cube
- One icon only

### Examples
- Owl (Granite Labs)
- Excel (Velixo)
- ChatGPT / AI symbol
- Chat bubble neural pattern

---

## 💡 Lighting System

- Primary glow: Cyan (#00D1FF range)
- Backlight bloom behind cube
- Edge highlights (thin, precise)
- Subtle floor reflection
- No overexposure

---

## ✍️ Typography System

### Top Label (Brand)
GRANITE LABS

- All caps
- White or light gray
- Slight letter spacing

---

### Main Title (Center)
[PRODUCT OR SYSTEM NAME]

Examples:
- CHATGPT AUTOMATION PLATFORM
- VELIXO REPORTING
- SYSTEM ARCHITECTURE PLATFORM

- Center aligned
- Clean sans-serif font
- Medium weight

---

### Bottom Line (Primary)
ACTION • ACTION • ACTION

Examples:
- PROMPT • GENERATE • AUTOMATE
- DATA • FINANCE • INSIGHT

---

### Bottom Line (Optional Secondary)
BUILD • STANDARDIZE • DEPLOY

- Smaller size
- Lower contrast

---

## ⚙️ Composition Rules

- Cube is always the focal point
- Centered composition preferred
- Balanced spacing throughout
- Text must not overpower visuals
- Use negative space intentionally

---

## 🚫 Restrictions

Do NOT:
- Use busy backgrounds
- Overuse cyan glow
- Add multiple icons
- Mix styles
- Use diagram-style layouts for OG images

---

## 📐 Export Standard

- Resolution: 1200 × 630
- Format: JPG
- Quality: 80–90
- Color: sRGB

---

## 📁 File Location

All OG images must be stored in:

/static/img/og/

---

## 🏷️ Naming Convention

Format:

[system]-[topic]-og.jpg

Examples:
- chatgpt-automation-platform-og.jpg
- velixo-reporting-og.jpg
- blueiris-system-architecture-og.jpg

---

## 🧠 Design Intent

This system ensures:

- Immediate brand recognition
- Visual consistency across all pages
- Clean, modern, technical presentation

OG images are for:
- Social previews
- Branding
- First impression visuals

NOT for:
- Technical diagrams
- Documentation graphics

---

## ✅ Success Criteria

A correct OG image:

- Matches GraniteLabs visual identity
- Uses cube-centered design
- Maintains clean typography
- Feels premium and consistent
- Is instantly recognizable
